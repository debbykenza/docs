## Que faire :

- Inscrivez-vous ou connectez-vous
- Allez dans la section **« Clés API »**.
- Cliquez sur **« Générer une clé API »**.
- Copiez la clé privée qui s’affiche 

<Warning>
  **Important **: chaque clé est unique pour l’utilisateur et **ne doit pas être partagée ni exposée **publiquement.
</Warning>

  <Step title="Utilisez la clé dans les requêtes HTTP" titleSize="h2">
    Exemple 1 – Projet JavaScript (React, Node.js…)

    <CodeGroup>

    ```javascript services/api.js or App.js
    import axios from "axios";
    
    const API_BASE_URL = "https://your-api.com"; // Your FastAPI base URL
    const API_KEY = process.env.API_KEY || "eyJhbGciOiJIUzI1NiIs..."; // API key
    
    export const getOrders = async () => {
      try {
        const res = await axios.get(`${API_BASE_URL}/commandes/`, {
          headers: {
            Authorization: `Bearer ${API_KEY}`
          }
        });
        return res.data;
      } catch (error) {
        console.error("API error:", error.response.data);
      }
    };
    
    ```

    </CodeGroup>

    Exemple 2 – Projet Python (Flask, FastAPI, etc.)

    <CodeGroup>

    ```javascript main.py
    import requests
    
    API_KEY = "eyJhbGciOiJIUzI1NiIs..."  # Your API key
    BASE_URL = "https://your-api.com"
    
    headers = {
        "Authorization": f"Bearer {API_KEY}"
    }
    
    response = requests.get(f"{BASE_URL}/livreurs/", headers=headers)
    
    print(response.status_code)
    print(response.json())
    
    ```

    </CodeGroup>
  </Step>
  <Step title="Tester l’API" titleSize="h2">
    <Check>
      **DOM API DOCUMENTATION**
    </Check>
    > Vous pouvez tester les points de terminaison directement dans le navigateur.

    #### Étapes:

    1. Allez dans votre **DOCUMENTATION API DOM**.
    2. Accédez à la section **api-reference**.
    3. Dans la barre latérale, choisissez une route (par exemple, **GET /commandes/**).
    4. Cliquez sur la route pour afficher les détails.
    5. Cliquez sur le bouton **« Try It »**.
    6. Saisissez tous les paramètres ou en-têtes requis (par exemple, **Authorization**).
    7. Cliquez sur **Envoyer la requête**.

    ![Api Reference Pn](/images/api_reference.png)

    La réponse apparaîtra en dessous avec le code de statut et les données.

    <Warning>
      Utilisez votre clé API dans l’en-tête comme suit :
    </Warning>
    ```json
    Authorization: Bearer <your_api_key>
    ```
  </Step>
</Steps>

<Check>
  **POSTMAN**
</Check>

> Postman est un outil populaire pour tester les API REST.

#### Étapes:

1. Ouvrez https://postman.com ou lancez l’application.
2. Cliquez sur **Nouveau → Requête HTTP**.
3. Saisissez l’URL de votre API, par exemple :

```
GET https://your-api.com/commandes/
```

1. Allez dans l’onglet **Autorisation** :
   - **Type** : Bearer Token
   - **Token** : votre_clé\_api
2. Cliquez sur **Envoyer**.

Vous verrez la réponse dans le panneau inférieur.

<Check>
  **SWAGGER UI (FastAPI)**
</Check>

> FastAPI inclut par défaut une interface interactive **Swagger UI**.

#### Étapes:

1. Si vous exécutez localement, allez à :

   <CodeGroup>

   ```bash
   http://localhost:8000/docs 
   ```

   </CodeGroup>

   Si vous êtes déployé sur Render :

   <CodeGroup>

   ```arduino
   https://your-api.onrender.com/docs
   ```

   </CodeGroup>
2. Tous vos points de terminaison sont listés. Cliquez sur l’un d’eux pour l’ouvrir.
3. Cliquez sur **« Authorize »**, puis saisissez :

   <CodeGroup>

   ```php-template
   Bearer <your_api_key>
   ```

   </CodeGroup>
4. Cliquez sur **« Try it out »**.
5. Remplissez les paramètres (le cas échéant) et cliquez sur **Exécuter**.

Vous verrez la requête complète et la réponse en dessous.
